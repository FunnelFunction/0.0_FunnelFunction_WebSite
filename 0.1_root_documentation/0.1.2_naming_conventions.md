# Naming Conventions

> **0.1.2** - Complete naming convention reference

---

## Core Pattern

```
[MAJOR].[MINOR].[PATCH]_[type]_[descriptor].ext
```

### Components

| Component | Description | Example |
|-----------|-------------|---------|
| **MAJOR** | Category/Domain (0-9) | `2` = Deep Knowledge |
| **MINOR** | Subcategory (0-9) | `2.1` = Whitepapers |
| **PATCH** | Specific item (0-9+) | `2.1.1` = First whitepaper |
| **type** | What kind of file | `content`, `script`, `config` |
| **descriptor** | What it does/contains | `mathematical_decomposition_of_trust` |
| **ext** | File extension | `.md`, `.py`, `.css` |

---

## Major Categories

| Number | Domain | Description |
|--------|--------|-------------|
| **0** | Root | Configuration, documentation, meta files |
| **1** | Pages | Core static pages (Home, About, Contact) |
| **2** | Deep Knowledge | Whitepapers, Research, Library, Textbooks |
| **3** | Articles | Blog posts, Medium, Substack aggregation |
| **4** | Social | Social media feeds, aggregator engine |
| **5** | Simulators | Interactive tools and marketing engines |
| **6** | Equations | Mathematical implementations, PyTorch models |
| **7** | Assets | Images, styles, fonts, static resources |
| **8** | Scripts | Build, deploy, utility scripts |
| **9** | Tests | Unit, integration, end-to-end tests |

---

## Type Prefixes

| Prefix | Usage | Example |
|--------|-------|---------|
| `index_` | Entry point, table of contents | `0.1.0_index_documentation.md` |
| `content_` | Prose, written content | `1.1.2_content_master_equation.md` |
| `config_` | Configuration file | `4.1.1_config_linkedin_api.json` |
| `script_` | Executable script | `4.9.1_script_master_aggregator.py` |
| `embed_` | Embeddable HTML widget | `4.2.2_embed_twitter_timeline.html` |
| `styles_` | CSS styling | `1.1.4_styles_home.css` |
| `util_` | Utility function | `8.3.1_util_wordpress_export.py` |
| `test_` | Test file | `9.1.1_test_equations.py` |
| `model_` | ML/PyTorch model | `6.1.3_model_commitment_function.py` |

---

## Folder Naming

Folders follow the same pattern but without type prefix:

```
[MAJOR].[MINOR]_[descriptor]/
```

### Examples

```
0.0_root_config/
0.1_root_documentation/
1.1_page_home/
2.1_whitepapers/
2.1.1_whitepaper_mathematical_decomposition_of_trust/
4.1_feed_linkedin/
5.1_simulator_campaign_commander/
6.2_equation_commitment_function/
```

## Special Cases

### Date-based content (blog posts)
```
3.1.1_post_YYYY-MM-DD_descriptor.md
3.1.1_post_2025-12-07_death_of_trust.md
```

### Nested depth (whitepapers with multiple files)
```
2.1.1.0_index.md          # Whitepaper index
2.1.1.1_content_abstract.md
2.1.1.2_content_body.md
2.1.1.3_equations.py
2.1.1.4_references.bib
```

## Anti-Patterns (Don't Do This)

| Bad | Why | Good |
|-----|-----|------|
| `readme.md` | No version number | `0.0.0_README.md` |
| `styles.css` | No category | `7.2.0_index_styles.css` |
| `my-script.py` | Hyphens, no prefix | `8.3.1_script_my_purpose.py` |
| `test1.py` | Meaningless name | `9.1.1_test_equations_unit.py` |
| `final_v2_FINAL.md` | Version chaos | Use git + semantic numbers |
